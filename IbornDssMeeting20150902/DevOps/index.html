Issues

  • Scenarios

1st scenario

  • Issue is reported
  • Estimated time: end of the day
  • 2 possible outcomes based on the priority
    Normal: Deployed in the next release - PROBLEM
    Immediate: Hotfix performed - PROBLEM

2nd scenario

  • Issue is reported (Normal priority)
  • Estimated time: 2-3 days
  • Meanwhile team of developers are developing priority project
  • Both are scheduled for the next release week
    PROBLEM: Issue depends on the status of priority project

3rd scenario

  • Issue is reported (Immediate priority)
  • Estimated time: 2-3 days
  • Meanwhile couple of minor fixes are also included
  • PROBLEM: The other fixes are not properly accepted, tested or implemented

4th scenario

  • Priority project is in progress
  • Scheduled for the next release week
  • Meanwhile, other issues are scheduled to be deployed
  • Project is rejected!
  • PROBLEM: There is no rollback
